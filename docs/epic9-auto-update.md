# Epic 9.3 — Auto-Update on Launch

## How it works

On app launch, the `UpdateChecker` component calls Tauri's updater plugin to check
for new versions. If an update is found, a slim banner slides down from the top of
the window showing the new version and an "Update now" button.

## Architecture

- **Backend**: `tauri-plugin-updater` + `tauri-plugin-process` (Rust plugins)
- **Frontend**: `UpdateChecker` component (`src/components/UpdateChecker.tsx`)
- **Endpoint**: GitHub Releases — `latest.json` is auto-generated by `tauri-action`

## Update flow

1. App starts, `UpdateChecker` mounts
2. Calls `check()` from `@tauri-apps/plugin-updater`
3. If update found: shows banner with version + "Update now" button
4. User clicks "Update now": downloads with progress bar, installs, relaunches
5. User can dismiss the banner with the X button

## Configuration

In `src-tauri/tauri.conf.json`:

```json
"plugins": {
  "updater": {
    "pubkey": "<your-public-key>",
    "endpoints": [
      "https://github.com/marikadeveloper/wow-dmg-simulator/releases/latest/download/latest.json"
    ]
  }
}
```

## Required setup before first release

1. **Generate signing keypair**:
   ```bash
   pnpm tauri signer generate -- -w ~/.tauri/wow-topgear.key
   ```
2. **Put the public key** in `tauri.conf.json` → `plugins.updater.pubkey`
3. **Add GitHub Secrets**:
   - `TAURI_SIGNING_PRIVATE_KEY` — content of the private key file
   - `TAURI_SIGNING_PRIVATE_KEY_PASSWORD` — password (or empty string)

Without these, updater artifacts won't be signed and auto-update won't work.

## Dev mode behavior

The updater silently does nothing in dev mode:
- `window.__TAURI__` check prevents crashes in browser-only `pnpm dev`
- `check()` is wrapped in try-catch so missing updater config doesn't error

## Build artifacts

With `createUpdaterArtifacts: true` in `tauri.conf.json`, the build produces:

| Platform | Updater artifact    | Signature          |
| -------- | ------------------- | ------------------ |
| macOS    | `.app.tar.gz`       | `.app.tar.gz.sig`  |
| Windows  | `-setup.exe` / `.msi` | `.exe.sig` / `.msi.sig` |

`tauri-action` uploads all of these plus `latest.json` to the GitHub Release.
